\doxysubsection{Detailed Description}
Shape Practice Solution. 

Using a Root Descriptor instead of Descriptor Table For performance, there is a limit of 64 DWORDs that can be put in a root signature.

Place all of the scene geometry in one big vertex and index buffer. Then use the Draw\+Indexed\+Instanced method to draw one object at a time ((as the world matrix needs to be changed between objects)

Controls\+: Hold down \textquotesingle{}1\textquotesingle{} key to view scene in wireframe mode. Hold the left mouse button down and move the mouse to rotate. Hold the right mouse button down and move the mouse to zoom in and out.

\begin{DoxyAuthor}{Author}
Hooman Salamat
\end{DoxyAuthor}
The three types of root parameters have the following costs \+:
\begin{DoxyEnumerate}
\item Descriptor Table \+: 1 DWORD =\texorpdfstring{$>$}{>} the application is expected to bind a contiguous range of descriptors in a descriptor heap
\item Root Descriptor \+: 2 DWORDs
\item Root Constant \+: 1 DWORD per 32 -\/ bit constant
\end{DoxyEnumerate}

Unlike descriptor tables which require us to set a descriptor handle in a descriptor heap, to set a root descriptor, we simply bind the virtual address of the resource directly.

There are 3 steps that need to be changed in three methods\+: Build\+Root\+Signature(), Draw(), Draw\+Render\+Items() to convert from Descriptor Table to Root descriptor
